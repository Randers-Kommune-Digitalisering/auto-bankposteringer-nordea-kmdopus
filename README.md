# Fællesoffentlig Bankintegration (FOBI)

## Formål

FOBI er udviklet for at automatisere og standardisere kontering og bogføring af banktransaktioner i danske kommuner.

Projektets kerneformål er at:
- Reducere manuel bogføring
- Øge datakvalitet og sporbarhed
- Skabe ensartethed i håndtering af indbetalinger
- Understøtte skalerbar og regelbaseret automatisering
- Danne grundlag for bredere standardisering i sektoren

FOBI er ikke blot et værktøj – det er et forsøg på at gentænke hvordan finansielle arbejdsgange kan struktureres mere intelligent.

---

## Problemet

I mange offentlige organisationer:

- Styres udbetalinger centralt og struktureret
- Men indbetalinger er fragmenterede og afhænger af borgerens eller virksomhedens handlinger
- Matchning sker ofte manuelt eller semi-manuelt
- Regler lever i hoveder, Excel-ark eller lokale praksisser

Dette medfører:

- Tidsforbrug
- Risiko for fejlkontering
- Manglende konsistens
- Svag mulighed for genbrug og skalering

FOBI adresserer netop dette spændingsfelt.

---

## Løsningen

FOBI implementerer:

- En regelbaseret matchningsmotor
- Strukturerede matchkriterier
- Standardiseret datamodel
- Deterministisk og auditérbar beslutningslogik
- Mulighed for gradvis udvidelse mod mere avanceret automatisering

Systemet gør kontering:

- Reproducerbar
- Testbar
- Transparent
- Skalerbar

---

## Værdi

### 1. Operationel effektivitet
Automatisering reducerer manuel behandling og frigiver ressourcer til kontrol og kvalitetssikring.

### 2. Øget sikkerhed
Standardiseret logik mindsker risiko for fejl og utilsigtede afvigelser.

### 3. Datakvalitet
Struktureret kontering skaber bedre grundlag for rapportering, analyse og styring.

### 4. Skalerbarhed
Regelbaseret struktur kan genbruges på tværs af organisationer og sektorer.

### 5. Strategisk perspektiv
Projektet kan fungere som fundament for en bredere diskussion om standardisering af betalings- og matchningsprocesser i den offentlige sektor.

---

## Designprincipper

- Stateless applikationslogik
- Klar separation mellem regeldefinition og eksekvering
- Auditérbar beslutningskæde
- Database som sandhedskilde
- Eksplicit validering og schemas

## Ambition

- At kunne tilbydes som fælles offentlig infrastruktur
- At understøtte høj compliance og revisionssikkerhed
- At være teknologisk robust og transparent
